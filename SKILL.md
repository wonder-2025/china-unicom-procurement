# 中国联通采购与招标网标讯抓取

## 概述

通过公开 API 从 `chinaunicombidding.cn` 抓取网络安全相关采购公告和采购结果，输出为结构化表格（格式与移动/电信日报一致）。

## API

**列表接口:** `POST https://www.chinaunicombidding.cn/api/v1/bizAnno/getAnnoList`

**请求体:**
```json
{
  "pageNo": 1,
  "pageSize": 50,
  "modeNo": "BizAnnoVoMtable",
  "annoName": "关键词"
}
```

**请求头:**
```
Content-Type: application/json;charset=UTF-8
Accept: application/json
User-Agent: Mozilla/5.0 ...
Referer: https://www.chinaunicombidding.cn/bidInformation
```

**详情接口:** `GET https://www.chinaunicombidding.cn/api/v1/bizAnno/getAnnoDetailed/{id}`

**详情页URL:** `https://www.chinaunicombidding.cn/bidInformation/detail?id={id}`


### 分片规则
- 企业微信webhook单条消息不超过4096字节
- 按section分片：采购结果1片、采购公告1片
- 每片包含完整表格头
- 0条时header+summary合并为1条发送

### ⚠️ 企业微信Webhook表格显示
**必须用 `markdown_v2` 类型**，普通 `markdown` 类型不支持表格语法：
```python
# ✅ 正确
payload = {"msgtype": "markdown_v2", "markdown_v2": {"content": table_content}}

# ❌ 错误（表格不显示）
payload = {"msgtype": "markdown", "markdown": {"content": table_content}}
```

## Webhook 推送格式

**⚠️ 企业微信 webhook 机器人必须使用 `markdown_v2` 类型才能显示表格！**

```python
payload = {
    "msgtype": "markdown_v2",
    "markdown_v2": {"content": table_content}
}
```

普通 `markdown` 类型不支持表格语法（`| col | col |`），会导致表格显示异常。
Hermes 的 `send_message` 走应用消息 API，普通 `markdown` 就支持表格，与 webhook 机器人不同。

## 常见问题

1. **API无响应**：检查网络连接，确认 `chinaunicombidding.cn` 可访问
2. **中标厂商显示"-"**：部分公告格式不规范，正则无法匹配
3. **遗漏标讯**：关键词列表可能未覆盖所有网安项目，根据实际情况调整
4. **假阳性**：建筑/交通安全项目被误匹配，更新EXCLUDE_WORDS列表
5. **API annoType 分类错误**：标题含「中选结果公示」但 API 返回 `annoType=采购公告` → 必须加标题关键词兜底（`if "中选结果" in title or "结果公示" in title`）
6. **ISO管理体系假阳性**：`ISO37301合规管理体系` 命中「合规」关键词但非网安 → EXC 需加 `ISO` `反贿赂` `管理体系认证` `合规管理体系` 等排除词
5. **API annoType 分类不可靠**（v1.4.2 — 2026-05-01）：API 可能将"中选结果公示"错标为"采购公告"，导致输出分类错误。修复：`main()` 分类时增加标题关键词兜底（"中选结果"/"结果公示"/"成交结果"/"中选候选人" → 强制归为采购结果）
6. **"合规"关键词过宽**（v1.4.2 — 2026-05-01）：匹配到 `ISO37301合规管理体系`、`ISO37001反贿赂管理体系` 等管理认证项目。修复：EXCLUDE_WORDS 新增 `ISO`、`反贿赂`、`管理体系认证`、`合规管理体系`
